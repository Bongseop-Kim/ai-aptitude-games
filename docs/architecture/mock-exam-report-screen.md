# 모의고사 리포트 화면 가이드

[mock-exam-report.md](./mock-exam-report.md)에서 정의한 데이터를 화면에 어떻게
보여줄지 정의한다. 대상 화면은 기록 탭 목록과 리포트 상세
(`src/screens/ReportDetailScreen.tsx`)다.

문구는 [voice-and-tone](../foundation/voice-and-tone.md)과
[writing](../foundation/writing.md), 로딩 표현은 [loading](../foundation/loading.md),
상태 표현은 [state](../foundation/state.md)를 따른다.

## 1. 정보 위계

리포트는 사용자의 3가지 질문에 순서대로 답한다.

| 순서 | 사용자 질문 | 담당 섹션 |
| --- | --- | --- |
| 1 | 몇 점이야? 지난번보다 나아졌어? | 커버(종합), 성장 추이 |
| 2 | 뭘 잘했고 뭘 보완해야 해? | 게임별 결과, 5대 역량, 강점·보완 Top 3, 면접 피드백 |
| 3 | 그래서 다음에 뭘 하면 돼? | AI 코치, 하단 CTA |

섹션 순서 (위 → 아래):

1. 커버 — 종합 점수·요약·이번 회차의 강점/보완 1개씩
2. 게임별 결과 — 9개 게임 실측치 (신설)
3. 5대 역량 프로필
4. 강점·보완 Top 3
5. AI 면접 피드백
6. 스트레스 복원력 (Pro)
7. 응답 패턴 프로필 (Pro)
8. 성장 추이 · 또래 비교
9. AI 코치 · 개선 플랜 (Pro)

핵심 결론(점수·등급·요약)은 항상 첫 화면(스크롤 전)에 들어온다.
세부 수치는 아래로 내려갈수록 깊어지고, 화면 어디서든 마지막은 행동 제안으로 끝난다.

## 2. 공통 상태 규칙

리포트의 모든 섹션은 아래 5개 상태를 가진다. 어떤 상태에서도 콘텐츠 높이가
변하지 않도록 실제 콘텐츠와 같은 높이를 예약한다(레이아웃 안정).

| 상태 | 조건 | 표현 |
| --- | --- | --- |
| 로딩 | 로컬 DB 조회 중 (1초 내외) | Skeleton. 실제 컴포넌트와 같은 크기 |
| 분석 중 | `mock_exam_reports.status != 'done'` | Skeleton이 아닌 **명시적 상태 카드**. 아이콘 + "결과를 분석하고 있어요" + 보조문구. 분석은 분 단위일 수 있으므로 진행 애니메이션을 무한 반복하지 않는다 |
| 분석 실패 | `status = 'failed'` | "리포트를 분석하지 못했어요. 잠시 후 다시 시도해주세요." + 다시 시도 버튼. 원시 기록은 남아 있음을 알린다 |
| 데이터 없음 | 또래 비교 값이 null 등 | 해당 표시를 **숨기고** 자기 비교로 폴백. 빈 자리나 "데이터 없음" 문구를 남기지 않는다 |
| Pro 잠금 | `section.locked && !record.pro` | 기존 `LockedReportSection` 패턴(흐림 + 잠금 오버레이). 문구는 혜택 설명형 — "복원력·응답 패턴·AI 코치는 Pro에서 열려요" |

분석 중 상태 카드 공통 형태:

```
┌─────────────────────────────────┐
│  ◷  결과를 분석하고 있어요       │
│     게임 기록을 바탕으로 역량을  │
│     계산하는 중이에요. 잠시 후   │
│     다시 확인해주세요.           │
└─────────────────────────────────┘
```

- 면접 분석(`report.interview.status`)은 게임 분석과 독립적으로 늦게 도착할 수 있다.
  이때 다른 섹션은 정상 표시하고 면접 섹션만 분석 중 상태를 유지한다.

## 3. 섹션별 가이드

### 3-1. 커버 (종합)

**목적**: 10초 안에 "이번 회차가 어땠는지" 결론을 준다.

```
6월 12일 · 22분 소요
오늘의 역량 지도
9개 게임 · N문항 · 5대 역량 분석     ← 실제 라운드 합계에서 파생

┌─────────────────────────────────┐
│  ◔ 76        종합 준비도         │
│   /100       71~81 범위          │  ← report.overall.score_range
│              "시간 압박 게임에서  │  ← report.overall.summary
│               안정적으로 수행"    │
│  [첫 회차 대비 +9]               │  ← 로컬 records 파생 (또래 배지 아님)
└─────────────────────────────────┘

┌── 강한 영역 ────┐ ┌── 보완 영역 ────┐
│ 신뢰 · 82       │ │ 관계 · 68       │  ← report.competencies 최고/최저
│ 책임감·윤리 판단 │ │ 협업·갈등 조율  │
└─────────────────┘ └─────────────────┘
```

- 종합 점수는 게이지(`ReadinessGauge`) + 큰 숫자. 화면에서 가장 큰 숫자는
  이 점수 하나뿐이어야 한다.
- 점수 옆 보조 표기는 **자기 비교**가 기본: "첫 회차 대비 +9".
  또래 백분위 배지는 `overall.percentile`이 null이 아닐 때만 추가한다
  (형식은 5장 수치 표기 규칙).
- `summary`는 can-do형 한 줄. 등급(band)을 그대로 노출하기보다 서술로 풀어 쓴다.
- 분석 중일 때: 게이지 자리에 로컬 점수(`mock_exam_results.score`)를 우선 표시하고
  "분석 중" 배지를 붙인다. 분석 완료 후 서버 점수로 갱신될 수 있음을 감안해
  점수 차이가 생겨도 레이아웃은 동일하다.

### 3-2. 게임별 결과 (신설)

**목적**: 회차를 구성한 9개 게임의 실측치를 한눈에. 서버 분석 없이도 항상 동작한다.

```
게임별 결과
┌──────────────────────────────────────┐
│ 개수 비교    ████████░░  88  ▲4      │
│              정확도 90% · 평균 1.2초  │
│ 마법약 만들기 ███████░░░  85  ▲2      │
│ ...                                  │
│ 숫자 누르기  █████░░░░░  63  ▼5      │
│              "6자리부터 정답률이      │  ← report.games[].insight
│               떨어졌어요"             │     (없으면 숨김)
└──────────────────────────────────────┘
```

- 행 구성: 게임명 | 점수 막대 | 점수 | 직전 회차 대비 ▲▼ (이전 기록 없으면 생략).
- 보조 행: 정확도(%)와 평균 반응시간(초). 두 값은 로컬 `game_results`에서 읽는다.
- `insight`가 있는 게임만 한 줄 코멘트를 붙인다. 9개 전부에 코멘트를 강제하지 않는다.
- 정렬은 모의고사 진행 순서 고정. 점수순 정렬은 Top 3 섹션이 담당하므로 중복하지 않는다.
- 행 탭 → 해당 게임 단독 훈련 진입(딥링크). 탭 가능함을 ChevronRight로 표시한다.

### 3-3. 5대 역량 프로필

**목적**: 게임 수행을 역량 언어(신뢰·전략·관계·가치·조직적합)로 번역해 보여준다.

```
┌──────────────────────────────────────┐
│ 신뢰    ──────●──── 82   상위 12%    │
│ 전략    ────●────── 76   상위 28%    │
│ 관계    ──●──────── 68   상위 45%    │
│            ▲                         │
│            └ 또래 중앙값 마커         │
└──────────────────────────────────────┘
```

- 기본 표현은 **불릿형 행**: 역량명 | 점수 막대 + 또래 중앙값 마커(`peer_median`) | 점수 | 백분위.
- 수치 판독은 막대가 담당한다. 레이더 차트는 유지하더라도 막대 행 위에
  "형태 인상"용 보조로만 두고, 레이더에서 수치를 읽게 만들지 않는다.
- `peer_median`·`percentile`이 null이면 마커와 백분위 열을 통째로 숨긴다
  (막대 + 점수만 남아도 성립하는 레이아웃).
- 역량 모델 설명("검증된 역량 모델" 등)은 실제 검증 근거가 payload에 실리기 전까지
  쓰지 않는다.

### 3-4. 강점 · 보완 Top 3

**목적**: "뭘 잘하고 뭘 보완할까"에 순위로 답하고, 보완 항목은 행동으로 연결한다.

```
강점 Top 3
1  개수 비교        Subitizing            88
   크기 착시를 억제하고 개수만 추정했어요

보완 Top 3
1  숫자 누르기      Digit Span 역순        63
   6자리 이상에서 정답률이 낮아졌어요
   [이 게임 5분 훈련하기]                  ← highlights.growth_areas[].action
```

- 순위(1·2·3)가 주인공이다. 점수는 우측에 보조로 둔다.
- 강점 행은 관찰 노트로 끝나도 된다. **보완 행은 반드시 행동 버튼으로 끝난다**
  (`action.game_id`로 훈련 딥링크, `action.minutes`로 소요시간 표기).
- 노트 문구는 능력 단정이 아닌 관찰: "정답률이 낮아졌어요" (O),
  "기억력이 부족해요" (X).

### 3-5. AI 면접 피드백

**목적**: 면접 답변을 5축(내용·구조·음성·시선·전달력)으로 분해하고
문항 단위로 복기하게 한다.

표시 단계가 둘로 나뉜다:

| 단계 | 조건 | 표시 |
| --- | --- | --- |
| 실측 단계 | `report.interview.status = 'pending'` | 문항 수, 문항별 답변 시간 vs 권장 시간, 재응답 횟수 + "답변 분석을 준비하고 있어요" 상태 카드 |
| 분석 단계 | `status = 'done'` | 아래 전체 구성 |

분석 단계 구성 (위 → 아래):

1. 총평 카드 — `overall_score` + 등급(`band`: 부족·필요·우수·완성) 4단계 스텝 표시
2. 5축 막대 — 축별 점수 + 또래 평균(`peer_avg`, null이면 숨김).
   `gaze`/`delivery`가 null(영상 분석 전)이면 해당 축을 잠금이 아닌 "준비 중" 처리
3. NCS 능력단위 평가 — 라벨 + 막대 + 점수 행
4. 가장 먼저 고칠 점 — `top_fixes` 최대 3개. 카드당 하나의 교정 행동
5. 문항별 아코디언 — 질문 | 축별 점수 | 전사(`transcript`) | 잘한 점/고칠 점.
   전사는 접힌 상태가 기본(법적·정서적으로 민감한 콘텐츠이므로 사용자가 펼친다)

### 3-6. 스트레스 복원력 (Pro)

**목적**: 난이도 상승·오답 직후 구간에서 수행이 어떻게 변했는지 보여준다.

- 라인 차트: x축 = 게임×난이도 구간(`resilience.curve`), y축 = 수행도.
  구간 라벨은 "G5" 같은 코드가 아니라 게임명 축약으로 표기한다.
- 차트 아래 인사이트 타일 2개(`insights`): 하락 구간(warning) + 회복(positive).
  서버 텍스트를 그대로 렌더링하며 클라이언트에서 가공하지 않는다.
- `resilience`가 null이면(P2 전) 섹션 전체를 분석 중 상태로 둔다.

### 3-7. 응답 패턴 프로필 (Pro)

**목적**: 점수가 아닌 "사고 스타일"을 보여준다. 좋고 나쁨이 없는 축임을 명확히 한다.

```
신중함   ──────●───   직관
속도     ───●──────   정확도
```

- 양극 스케일 행(`response_pattern.scales`): 좌/우 라벨 + 위치 마커.
- 마커에 색상 점수 톤(positive/critical)을 쓰지 않는다 — 우열이 아니라 성향이다.
- 각 축 아래 한 줄 해석이 오면 표시하고, 없으면 라벨만으로 둔다.

### 3-8. 성장 추이 · 또래 비교

**목적**: "어디쯤 와 있을까"에 자기 추세 → 또래 위치 순서로 답한다.

1. **회차별 성장 추이** (항상 표시) — 라인 차트(로컬 records 점수) +
   "첫 회차 대비 +N점 성장했어요" 콜아웃. 기록이 1개면 차트 대신
   "다음 회차를 완료하면 성장 추이를 볼 수 있어요"를 표시한다.
2. **또래 비교** (`overall.percentile` 있을 때만) — 분포 바 + 내 위치 마커 +
   코호트 표기. 코호트 N과 라벨은 payload 값을 그대로 쓴다:
   "전체 사용자 1,240명 기준". 임의 숫자를 만들지 않는다.
3. **역량별 백분위** (`competencies[].percentile` 있을 때만) — 3-3과 같은 불릿 행.

### 3-9. AI 코치 · 개선 플랜 (Pro)

**목적**: 리포트의 결론. "다음 2주 동안 뭘 하면 되는지"로 끝난다.

- 인사이트 카드: `coach.insight` (title 한 문장 + body 한 문장). 두 문장을 넘기지 않는다.
- 훈련 플랜 리스트: 기간 배지(`day_range`) | 게임명 | 난이도 라벨 | 분/일.
  각 행은 해당 게임 훈련으로 딥링크. 마지막 행(모의고사 재도전)은 모의고사 시작으로 연결.
- 알림 리마인드 카드는 알림 기능이 실제로 연결되기 전까지 렌더링하지 않는다.

### 3-10. 하단 고정 CTA

화면 맨 아래 `BottomActionBar`에 단일 CTA 하나만 둔다.

- 기본: "보완 게임 5분 훈련하기" — `growth_areas[0].action`으로 연결
- 분석 중: "다음 모의고사 시작하기"
- 버튼이 두 개 이상 필요해 보이면 우선순위를 다시 정한다. CTA는 항상 하나다.

## 4. 기록 탭 (목록 화면)

상세로 진입하기 전, 목록은 "축적"을 보여주는 화면이다. 현재 구조를 유지한다.

- 요약 카드: 완주 회차 수 + 첫 회차 대비 delta + 추세 미니 라인(기록 2개 이상).
- 회차 행: 회차 번호 | 점수(75 이상 positive 톤) | 날짜 | 소요시간 |
  직전 대비 ▲▼ | Pro·최신 배지.
- 행의 점수는 분석 완료 시 서버 점수로 갱신된다. 분석 전 행에는 점수 옆에
  작은 "분석 중" 표시를 둘 수 있으나, 행 높이는 동일하게 유지한다.

## 5. 수치 표기 규칙

| 값 | 표기 | 예 |
| --- | --- | --- |
| 점수 | 정수, 컨텍스트로 `/100` 1회만 | 76 /100 |
| 점수 범위 | `N~M 범위`, 커버에서만 | 71~81 범위 |
| 백분위 | `상위 N%` + 코호트 병기 | 상위 28% · 전체 사용자 1,240명 |
| 회차 대비 변화 | ▲/▼ + 절대값, positive/critical 색상 + 기호 병행(색상 단독 금지) | ▲ 4 |
| 정확도 | 정수 % | 정확도 90% |
| 반응시간 | 소수 1자리 초 | 평균 1.2초 |
| 소요시간 | 분 단위 반올림 | 22분 소요 |
| 코호트 표본 | 천 단위 구분 쉼표, 실제 값만 | 1,240명 |

- 백분위·표본·상관계수 등 통계 수치는 payload에 실린 실제 값만 표시한다.
  payload에 없는 수치를 화면 코드에 쓰지 않는다.
- 같은 값은 화면 안에서 한 가지 형식으로만 표기한다(점수를 어디서는 등급,
  어디서는 %로 바꾸지 않는다).

## 6. 차트 선택 규칙

| 표현 의도 | 차트 | 비고 |
| --- | --- | --- |
| 항목 간 수량 비교 (역량, NCS, 5축) | 가로 막대 / 불릿 행 | 수치 텍스트 병기 필수 |
| 또래 기준 대비 내 위치 | 막대 + 중앙값 마커, 분포 바 + 위치 마커 | 마커 없는 막대로 강등 가능(null 폴백) |
| 시간 추세 (회차, 게임 내 구간) | 라인 | 포인트 7개 이하면 점 표시, 이상이면 선만 |
| 성향 (우열 없는 축) | 양극 스케일 행 | 점수 톤 색상 금지 |
| 다축 형태 인상 | 레이더 (보조 전용) | 단독 사용 금지, 항상 막대 행과 동행 |

공통:

- 모든 차트는 인접한 텍스트(수치·라벨)만으로도 같은 결론을 얻을 수 있어야 한다.
  차트는 보조 수단이고, 스크린리더는 텍스트를 읽는다.
- 색상만으로 의미를 전달하지 않는다. 기호(▲▼)·라벨·위치를 병행한다.
- 차트 컨테이너는 고정 높이. 데이터 개수가 변해도 높이가 출렁이지 않는다.

## 7. 리포트 카피 가이드

[voice-and-tone](../foundation/voice-and-tone.md) 원칙 위에 리포트 특화 규칙을 더한다.

1. **관찰 → 해석 → 행동 순서로 쓴다.** 점수만 던지고 끝나는 문장을 만들지 않는다.
2. **수행을 말하고 사람을 평가하지 않는다.** 주어는 "이번 회차·이 게임에서의 수행"이다.
3. **보완 문구는 과제 중심으로.** "당신은 약하다"가 아니라 "이렇게 하면 올라간다".
4. **단정 대신 경향.** "~했어요", "~하는 경향이 있었어요". "~입니다" 단정형 금지.
5. **분석 중·실패 상태도 같은 톤.** 기술 용어(서버, 동기화, 분석 파이프라인) 노출 금지.

| 상황 | Do | Don't |
| --- | --- | --- |
| 보완 영역 | 6자리 이상에서 정답률이 낮아졌어요. 5자리 훈련부터 다시 쌓아보세요. | 작업기억이 부족합니다. |
| 또래 비교 없음 | (또래 카드 숨김, 자기 비교만 표시) | 비교 데이터가 없습니다. |
| 분석 중 | 결과를 분석하고 있어요. 잠시 후 다시 확인해주세요. | 서버에서 리포트 생성 중... |
| 면접 분석 전 | 답변 분석을 준비하고 있어요. 답변 시간은 먼저 확인할 수 있어요. | 분석 실패 |
| 성장 콜아웃 | 첫 회차 대비 9점 성장했어요. | 상위권 진입까지 3점 남았습니다! |
